Factory Pattern

Create objects without specifying the exact class. Decouples creation logic from usage. Practical examples, UML, and pros/cons included.

When to Use

  • When the class cannot anticipate which concrete class to instantiate.
  • Decoupling creation logic from clients for maintainability.
  • Centralized object creation reduces duplication and errors.
Tip: Can combine with Singleton if only one factory instance is needed.

Checklist

  • Define a common interface for products
  • Create concrete classes implementing the interface
  • Implement a factory class/method to return products
  • Clients depend on interface, not concrete classes

Idiomatic Implementations

interface Shape { void draw(); }
class Circle implements Shape { public void draw(){ System.out.println("Circle"); } }
class Square implements Shape { public void draw(){ System.out.println("Square"); } }
class ShapeFactory {
  public static Shape create(String type){
    return switch(type){
      case "circle" -> new Circle();
      case "square" -> new Square();
      default -> throw new IllegalArgumentException();
    };
  }
}
Shape s = ShapeFactory.create("circle");
s.draw();
interface IShape { void Draw(); }
class Circle : IShape { public void Draw() => Console.WriteLine("Circle"); }
class Square : IShape { public void Draw() => Console.WriteLine("Square"); }
class ShapeFactory {
  public static IShape Create(string type) =>
    type switch {
      "circle" => new Circle(),
      "square" => new Square(),
      _ => throw new ArgumentException()
    };
}
var s = ShapeFactory.Create("circle");
s.Draw();
class Circle { draw(){ console.log("Circle"); } }
class Square { draw(){ console.log("Square"); } }
class ShapeFactory {
  static create(type){
    return type === "circle" ? new Circle() : new Square();
  }
}
const s = ShapeFactory.create("circle");
s.draw();
class Circle:
    def draw(self): print("Circle")
class Square:
    def draw(self): print("Square")
class ShapeFactory:
    @staticmethod
    def create(type):
        if type == "circle": return Circle()
        elif type == "square": return Square()
        else: raise ValueError()
s = ShapeFactory.create("circle")
s.draw()

Pros & Cons

✅ Benefits

  • Decouples creation from usage
  • Easier to add new product types
  • Centralizes creation logic

⚠️ Pitfalls

  • Many small classes
  • Complex factory hierarchy
  • Overengineering for simple cases

Alternatives

  • Dependency Injection
  • Prototype Pattern
  • Static factory functions
