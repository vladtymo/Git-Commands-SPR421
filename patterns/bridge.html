Design Pattern • Structural

Bridge Pattern

Decouple abstraction from implementation so both can evolve independently.

When to Use

  • When you want to avoid a permanent binding between abstraction and implementation.
  • When both abstractions and implementations should be extensible independently.
  • When you want cleaner, more scalable architecture.

Implementation Examples


interface Device {
    void turnOn();
    void turnOff();
}

class TV implements Device {
    public void turnOn() { System.out.println("TV ON"); }
    public void turnOff() { System.out.println("TV OFF"); }
}

abstract class Remote {
    protected Device device;
    public Remote(Device device) {
        this.device = device;
    }
    public abstract void togglePower();
}

class BasicRemote extends Remote {
    public BasicRemote(Device device) {
        super(device);
    }
    public void togglePower() {
        device.turnOn();
    }
}

public class Demo {
    public static void main(String[] args) {
        Device tv = new TV();
        Remote remote = new BasicRemote(tv);
        remote.togglePower();
    }
}


interface IDevice {
    void TurnOn();
    void TurnOff();
}

class TV : IDevice {
    public void TurnOn() => Console.WriteLine("TV ON");
    public void TurnOff() => Console.WriteLine("TV OFF");
}

abstract class Remote {
    protected IDevice device;
    protected Remote(IDevice device) {
        this.device = device;
    }
    public abstract void TogglePower();
}

class AdvancedRemote : Remote {
    public AdvancedRemote(IDevice device) : base(device) {}
    public override void TogglePower() {
        device.TurnOn();
    }
}

class Program {
    static void Main() {
        IDevice tv = new TV();
        Remote remote = new AdvancedRemote(tv);
        remote.TogglePower();
    }
}


class Device {
    turnOn(){}
    turnOff(){}
}

class TV extends Device {
    turnOn(){ console.log("TV ON"); }
    turnOff(){ console.log("TV OFF"); }
}

class Remote {
    constructor(device){
        this.device = device;
    }
    togglePower(){
        this.device.turnOn();
    }
}

const tv = new TV();
const remote = new Remote(tv);
remote.togglePower();

Pros & Cons

Advantages

  • Independent extensibility
  • Cleaner architecture
  • Prevents class explosion

Disadvantages

  • More abstraction layers
  • Can complicate small projects
