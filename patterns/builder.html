The Builder Pattern

A step-by-step approach to constructing complex objects. It separates the construction of an object from its representation, allowing the same process to create different versions.

Why use the Builder?

Imagine a complex object that requires step-by-step initialization of many fields and nested objects. Such initialization code is usually buried inside a monstrous constructor with dozens of parameters.

  • Solves "Telescoping Constructor" anti-pattern (too many parameters).
  • Allows for step-by-step construction.
  • Provides better control over the construction process.
  • Keeps the product class clean and focused on data/behavior.

Checklist

  • Identify complex product
  • Create Builder class
  • Implement step methods
  • Add build() method
  • Optional: Create Director
Pro Tip: Use the Builder pattern when your constructor has more than 4-5 parameters.

Structure Overview

Client → Director → Builder → Product

The Director defines the order, the Builder provides implementation.

Implementation

Fluent Interface Builder

public class Computer {
    private String CPU, RAM;
    private boolean isGraphicsCardEnabled;

    public static class Builder {
        private Computer comp = new Computer();

        public Builder setCPU(String cpu) { comp.CPU = cpu; return this; }
        public Builder setRAM(String ram) { comp.RAM = ram; return this; }
        public Computer build() { return comp; }
    }
}
// Usage: new Computer.Builder().setCPU("i9").setRAM("32GB").build();

C# Implementation

public class Pizza {
    public string Dough { get; set; }
    public string Topping { get; set; }
}

public interface IPizzaBuilder {
    void Reset();
    void SetDough();
    void SetTopping();
    Pizza GetProduct();
}

Modern JavaScript (ES6)

class UserBuilder {
    constructor(name) {
        this.user = new User(name);
    }
    setAge(age) {
        this.user.age = age;
        return this;
    }
    build() { return this.user; }
}
const user = new UserBuilder('Alice').setAge(25).build();

Pythonic Way

class HouseBuilder:
    def __init__(self):
        self.house = House()
    
    def add_walls(self):
        self.house.walls = 4
        return self

    def build(self):
        return self.house
